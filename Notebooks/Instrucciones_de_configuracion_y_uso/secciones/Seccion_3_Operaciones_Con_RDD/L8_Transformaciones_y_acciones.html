8. Transformaciones y acciones

Operaciones con RDDs.

Fecha: 15 de enero, 2021.

🎗️Recordatorio.

Es importante tener claro el tipo de la estructura (RDD o DataFrame) durante el desarrollo, ya que hay operaciones que solo pueden tener efectos en RDDs o en DataFrames.

Nota: En clases posteriores explicaremos el motivo de la restricción del uso del método .collect() para visualizar el contenido de un data set.

Hasta que no realicemos una acción, no seremos capaces de ver los errores de los RDDs

Recuerda detener el(los) contexto(s) para liberar la memoria. Para detener el contexto, ejecuta la linea sc.stop().

Según la experiencia del profesor, la carga de archivos hacia un RDD no es una práctica común.

Siempre debes verificar las rutas de los archivos en Anaconda, para ejecutar comandos de Linux en Anaconda, debemos usar el prefijo !

📒Notas.

Lista de métodos sobre RDDs:

  • rdd1 = sc.parallelize([1,2,3]) Esto significa que la variable rdd1 es un dataset (una lista) asignada al contexto sc con el método parallelize y con el argumento lista.
  • rdd1.collect() Esto permite visualizar el contenido de la variable rdd1.
  • Recordemos que la variable rdd1 es del tipo RDD, para ver el tipo de una variable escribimos type(rdd1)

La primera impresión de este RDD podría ser una simple lista de tres elementos, sin embargo podemos ver en la interfaz del usuario de Spark que esta estructura (rdd1) tiene más atributos:

  • En la siguiente pieza de còdigo observamos varios métodos sobre la variable equiposOlimpicosRDD.
path = "/home/cleto/Desktop/Intro_to_Spark/files/"
equiposOlimpicosRDD = sc.textFile(path + "paises.csv")\
.map(lambda line : line.split(","))

El método .textFile convierte un archivo de texto en un RDD (el argumento paises.csv, que habita dentro del directorio path). Para poder hacer saltos de linea y dividir el RDD en columnas, debemos hacer uso del método .map y de la función lambda sobre los separadores del archivo csv (que por defecto son comas).

  • El método .take(n) (donde n es un numero entero positivo y representa la cantidad de lineas que se expresarán) es una alternativa para visualizar parcialmente el contenido de un RDD.
