Recordemos que Spark se desempeña principalmente en la administración de grandes volúmenes de datos ubicados en un warehouse. Para fines de este curso, ese warehouse será la máquina local, en donde habitan un driver y un ejecutor.

RDD (Resilient Distributed Dataset).

Es una estructura nativa de Spark que pueden set creados al cargar datos de manera distribuida, como desde un HDFS (Hadoop Distributed File System), Cassandra (un manejador No-SQL creado por Apache), Hbase o cualquier sistema de datos soportado por Hadoop, pero también por colecciones de datos de Scala o Python, a parte de ser leídos desde archivos en el sistema local.

De manera muy general, se podría decir que un RDD puede considerarse como un set de datos que solo pueden soportar dos tipos de operaciones: transformaciones y acciones.

Las transformaciones permiten crear un nuevo RDD a partir de uno previo, mientras que las acciones retornan un valor al driver. El núcleo de operación del paradigma de Spark es la ejecución perezosa (Lazy). Esto quiere decir que las transformaciones serán computadas posterior a la llamada de la acción.

Los RDD poseen algunas similitudes con el paradigma POO, lo que les dotan de operatividad de bajo nivel.

La gran ventaja del uso de los RDDs es la compilación segura (producto de la ejecución lazy), esta propiedad permite identificar problemas antes de lanzar la aplicación. Por otro lado, la desventaja del uso de los RDDs es que no son debidamente tratados por el Garbage Collector y cuando las operaciones se escalan (es decir, se hacen más complejas), su uso tiende ser impráctico.

DataFrames.

Estas estructuras fueron agregadas en la versión 1.3 de Spark, y pueden ser invocados con el contexto especial de Spark SQL. Este modulo (Spark SQL) está especialmente desarrollado para ser ejecutado con instrucciones similares al entandar SQL.

Los DataFrames, al igual que los RDDs, pueden ser creados a partir de archivos, tablas de tipo Hive, bases de datos externas y de RDDs o de otros DataFrames ya existentes.

Un DataFrame en Spark està compuesto de columnas nombradas, lo que le proporciona cierta similitud con los DataFrames de Pandas en cuanto a la manera de trabajar se refiere. Cada columna del DataFrame de Spark es considerada con ssu propio tipo de dato (Row).

Pero no es todo, los DataFrames implementan un sistema llamado Catalyst, el cual es un motor de optimización de planes de ejecución, parecido al que usan las bases de datos, pero diseñado para la cantidad de datos propia de Spark, aunado a eso, se tiene implementado un optimizador de memoria y consumo de CPU llamado Tungsten, el cual determina cómo se convertirán los planes lógicos creados por Catalyst a un plan físico.
