5. Jupyter vs CLI: ejecución de Spark desde la línea de comando.

Configuración.

Fecha: 12 de enero, 2021.

🎗️Recordatorio.

En Linux, los archivos invisibles empiezan con un punto en el nombre.

Los comentarios en el archivo .bashrc (y en la mayoría de los archivos de configuración), los comentarios se hacen con #

Para apuntar a una variable en un archivo de configuración, se utiliza el signo $

Las variables de entorno se identifican con mayúsculas, y no se permiten espacios innecesarios dentro de la asignación de variables.

Es posible que sucedan errores después de refrescar los cambios en el archivo .bashrc Si ya revisaste la edición y todo está bien, entonces cierra la terminal y vuelve a abrir.

Si requerimos correr Spark en un modo de ejecución en vivo, entonces debemos ejecutar pyspark.

Si requerimos correr Spark de manera directa, entonces ejecutamos spark-submit e indicamos el archivo *.py

Con el comando head -n 10 <file>

puedo ver las primeras 10 lineas del archivo <file>

📒Notas.

En esta clase ejecutaremos el primer proceso de Spark desde la linea de comandos en Ubuntu.

  • 1. Debemos configura algunas variables de entorno para poder hacer utilizable Spark sobre la linea de comandos.

    1.1 Posicionarse en la carpeta de usuario (cd ~), y editar el archivo .bashrc con el comando nano .bashrc

    1.2 En las ultimas lineas del archivo, escribir las siguientes lineas de texto: 

    ##Path de Java

export JAVA_HOME="/usr/lib/jvm/java-8-openjdk-amd64"

export PATH=$JAVA_HOME/bin:$PATH

##SPARK

export SPARK_HOME="/home/<user>/spark"
export PATH=$SPARK_HOME/$PATH
export PYTHONPATH=$SPARK_HOME/python:$PYTHONPATH
export PYSPARK_PYTHON=python3.7

    Nota: Reemplazar <user> por el usuario.

    1.3 Después de editar el archivo, guardarlo y refresca desde terminal con el comando source .bashrc

  • 2. Posterior a la ediciòn del archivo .bashrc, debemos explorar la carpeta de Spark.

    2.1 En la terminal, dirigirse a la carpeta de spark con el comando cd spark/bin/ , luego explorar el contenido con ls

    2.2 Ejecutamos el siguiente comando para correr un proceso con Spark desde CLI:

    #Posicionado en el directorio donde habitan 
#los scripts de python(*.py) y el dataseet(*.csv).
#En este caso particular, los archivos estàn en Documents

../spark/bin/spark-submit <pyscript.py> <data.csv>

El inconveniente de correr Spark por linea de comandos, es que se expresan todas las transacciones y operaciones entre el framework y los archivos, por lo que los productos que requerimos quedan enterrados en información que no es útil para el proceso final.
